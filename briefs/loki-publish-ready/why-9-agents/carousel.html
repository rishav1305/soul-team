Why Multi-Agent Systems Fail
1
Shared state conflicts — 2 agents edit the same thing
2
Contradictory decisions — agents give conflicting advice
3
Priority deadlocks — everyone waits for everyone else
Enterprise equivalent: 9 developers with no git, no Jira, and no standup.
Rishav Chatterjee | Senior AI Architect
Pattern 1: Inbox Protocol
Async messaging without shared state
◆ Each agent has a persistent inbox
◆ Messages are files: briefs, requests, corrections
◆ Agents process on their own schedule
◆ No race conditions, full audit trail
LinkedIn built their multi-agent coordination on existing messaging infrastructure. — InfoQ, Sept 2025
Rishav Chatterjee | Senior AI Architect
Pattern 2: Conference Protocol
1
Decision needed
Any agent can convene a conference
↓
2
2–5 agents convened
Each contributes domain expertise
↓
3
Consensus document
Action items, aligned direction
"Should we reposition the brand?" → Strategy + Content + Technical aligned in minutes, not weeks.

Pattern 3: Skill Sharing
65+
shared skills across all 9 agents
Skills are ABILITIES
"How to audit SEO"
"How to run analysis"
"How to debug a build"
Not DATA
"Last audit results"
"Latest output"
"Previous fix"
Any agent invokes any skill (access scoped by role). Prevents silos. Accelerates cross-domain work.
Rishav Chatterjee | Senior AI Architect

Machine 2: High-Compute
Heavy CPU/memory, on-demand
3 agents: code, data science, analytics

What Happens When Agents Disagree
1
Self-solve
Check skills, knowledge sources, web search
↓
2
Peer route
Send to specialist's inbox, continue other work
↓
3
Human escalate
Full context: what tried, what needed, what failed
The escalation chain IS the product. Without it: deadlocks or silent bad decisions.
